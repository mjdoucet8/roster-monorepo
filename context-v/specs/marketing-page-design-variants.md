# Marketing Page Design Variants

Status: Draft

## Purpose

Define three design variants for the first Roster marketing page. Each variant should communicate the same core product idea while exploring different visual, structural, and positioning directions.

Roster is a sports networking application that connects athletes and coaches while giving them tools to organize their teams.

## Shared Page Requirements

These requirements should apply to all three variants unless changed later.

- The page should clearly introduce Roster in the first viewport.
- The page should make the athlete-coach connection value obvious.
- The page should include team organization as a meaningful product pillar.
- The page should have one primary call to action.
- The page should be suitable for an early-stage product before the full application is built.
- The page should be implementable with static HTML and CSS only.
- The page should not depend on JavaScript, a frontend framework, backend data, or live app integrations.
- The primary CTA should be "Download app" or a close variant.
- The content should primarily speak to coaches onboarding and organizing teams, while still making athletes feel like direct participants in the network.

## Variant 1

Working name:

```text
Command Center
```

Audience angle:

```text
Coaches who need one place to onboard athletes, manage their roster, and keep the team moving.
```

Visual direction:

```text
Dark SaaS dashboard energy with athletic contrast. The page should feel organized, credible, and operational, while still using youthful social-app styling through sharp color accents, profile tiles, notification badges, and mobile-first visuals.
```

Page structure:

```text
1. Full-viewport hero with a coach-focused headline, app download CTA, and a large product mockup showing roster, schedule, and notification surfaces.
2. Three-feature band covering athlete profiles, team rosters, and schedules.
3. Coach workflow section showing onboarding athletes, organizing the roster, and keeping everyone updated.
4. Athlete network section showing that each athlete profile becomes part of the broader Roster ecosystem.
5. Final download CTA.
```

Hero message:

```text
Build your team network in one place.

Roster helps coaches onboard athletes, organize teams, and keep schedules, profiles, and notifications connected from day one.
```

Primary CTA:

```text
Download app
```

Key sections:

```text
1. Team command center: roster, schedule, and communication previews.
2. Athlete profiles: each player gets a profile coaches can review and athletes can build on.
3. Notifications that reach the team: email, text, and mobile updates.
```

Notes:

```text
This should be the most practical and coach-forward variant. It should feel like the safest first marketing direction if the goal is converting coaches into early adopters. Static HTML/CSS should use mock interface panels rather than real product screenshots.
```

## Variant 2

Working name:

```text
The Locker Room Feed
```

Audience angle:

```text
Coaches who want their team to feel connected, visible, and easy to engage, with athletes motivated to participate because the experience feels social.
```

Visual direction:

```text
Dark social network style inspired by Discord and TikTok. Use vertical rhythm, profile cards, short update blocks, high-contrast accents, and a sense of motion through layout, layering, and typography. The page should feel youthful without becoming chaotic.
```

Page structure:

```text
1. Hero with a social feed-style composition showing athlete profile cards, team updates, schedule reminders, and coach announcements.
2. Social proof-style feature strip explaining how athletes, coaches, and teams connect.
3. Split section showing coach controls on one side and athlete-facing profiles on the other.
4. Schedule and notification section designed like a stream of upcoming moments.
5. Final mobile app download CTA.
```

Hero message:

```text
Your team, already in sync.

Roster gives coaches the tools to organize the team while giving athletes a profile-driven space that feels natural to use.
```

Primary CTA:

```text
Download app
```

Key sections:

```text
1. Team feed: announcements, schedule changes, and reminders in a social layout.
2. Player identity: athlete cards that highlight profile, role, and availability.
3. Coach-to-team reach: email, text, and mobile notifications from one update.
```

Notes:

```text
This should be the most youth-oriented variant. It should make Roster feel culturally current and familiar to athletes while still showing coaches that the product solves organization problems. Static HTML/CSS can imply feeds, cards, and mobile screens without interactive behavior.
```

## Variant 3

Working name:

```text
Profiles To Programs
```

Audience angle:

```text
Coaches building a program over time, where organized athlete profiles, team structure, and scheduling create a stronger network around the team.
```

Visual direction:

```text
Premium sports editorial mixed with modern app marketing. Dark base, strong photography or photo-like placeholders, bold typography, and polished athlete profile presentation. This version should feel more aspirational and brand-forward than the other two.
```

Page structure:

```text
1. Editorial hero with strong Roster branding, a coach-led value proposition, app download CTA, and a large athlete profile visual.
2. Profile-first section showing how athlete profiles become the foundation for teams and coach discovery.
3. Program organization section covering roster management, schedules, and notifications.
4. Network growth section showing how coach onboarding brings athletes into the broader platform.
5. Closing CTA with app store-style download treatment.
```

Hero message:

```text
Turn every roster into a network.

Roster helps coaches organize their teams while giving every athlete a profile that can grow beyond the season.
```

Primary CTA:

```text
Download app
```

Key sections:

```text
1. Athlete profiles: the permanent identity layer for players.
2. Team organization: rosters and schedules coaches can manage clearly.
3. Network effects: coaches bring teams in, athletes carry their profiles forward.
```

Notes:

```text
This should be the most brand-forward and aspirational variant. It should position Roster as more than a team tool: a long-term sports identity and networking platform. Static HTML/CSS should rely on strong layout, typography, and image placeholders rather than dynamic product behavior.
```

## Acceptance Criteria

- The completed spec defines three clearly distinct marketing page variants.
- Each variant has a named audience angle, visual direction, page structure, hero message, CTA, and key sections.
- Shared content and product assumptions are explicit.
- The spec gives enough direction to create static HTML/CSS wireframes or frontend mockups without returning to the original conversation.
- The implementation can be completed without JavaScript, package installation, build tooling, or backend services.
