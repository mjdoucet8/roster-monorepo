# issues

Use this folder for named problems, bugs, risks, and failure modes that deserve durable context.

An issue note should explain the observed problem, suspected cause, affected areas, reproduction details if available, attempted fixes, current status, and links to related work. This is not a replacement for an issue tracker; it is where richer context can live.

Create an issue when a problem is subtle, recurring, cross-cutting, or likely to matter again after the immediate fix.
